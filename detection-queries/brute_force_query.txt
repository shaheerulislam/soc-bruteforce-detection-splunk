Detection Name: Brute Force Authentication Attack (SIEM Rule)

SOC Context:
This detection identifies repeated failed login attempts against user accounts, indicating a potential brute-force attack targeting credential access.

Why This Matters:
Brute-force attacks are commonly used by threat actors to gain unauthorized access by guessing passwords. This aligns with MITRE ATT&CK technique T1110 (Brute Force).

Detection Logic (Splunk Query):

index=auth_logs sourcetype=linux_secure "Failed password"
| stats count by src_ip, user
| where count > 5
| sort - count

Alert Condition:
Trigger when failed login attempts exceed 5 within a short time window.

SOC Analyst Actions:
- Validate if source IP is known or malicious
- Check if multiple users are targeted
- Look for successful login after failed attempts
- Escalate to Tier 2 SOC if pattern indicates automation or high frequency attack

Severity Classification:
- Low: 3–5 attempts (user error likely)
- Medium: 5–10 attempts (suspicious behaviour)
- High: 10+ attempts or multi-user targeting (confirmed brute-force attack)

False Positive Considerations:
- Users mistyping passwords
- Service misconfiguration causing repeated authentication failures
- Automated internal scripts

SOC Value:
Demonstrates SIEM-based threat detection, log correlation, and Tier 1 SOC triage workflow.
